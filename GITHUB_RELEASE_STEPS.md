# iFicam GitHub Release and Auto-Update Setup

The app is configured to check GitHub Releases from:

https://github.com/dhruvdroid717/ificam

Installed users receive updates from the release files generated by GitHub Actions, especially `latest.yml` and the Windows installer `.exe` files.

## One-time GitHub setup

1. Create a new GitHub repository named `ificam` under the `dhruvdroid717` account.
2. Push this local project to that repository.
3. On GitHub, open `Settings > Actions > General`.
4. Under `Workflow permissions`, choose `Read and write permissions`.
5. Save.

## Push the first version

Run these commands from `D:\Playground\ificam` after signing in to GitHub:

```powershell
git push -u origin main
git tag v21.6.26
git push origin v21.6.26
```

The tag starts the release workflow at `.github/workflows/release.yml`.

## How future updates work

1. Change the app code.
2. Bump the app version in `package.json`.
3. Commit the change.
4. Push a new version tag.

Example:

```powershell
npm version patch
git push
git push origin v21.6.27
```

GitHub Actions will build and publish the installers. The bell icon inside iFicam checks that GitHub Release and lets users download/install the update.

## Notes

- `version` must be valid SemVer, like `21.6.27`.
- `buildVersion` can keep your display/build style, like `21.06.27.1`.
- If the GitHub username or repo changes, update `publish.owner` and `publish.repo` in both `package.json` and `electron-builder.yml`.
